**Ранжирование и процентная динамика**

Загрузка с файла в датафрейма Pandas (itresume-coderun.csv).

Создание в датафрейме столбца rnk - ранжирование строки по заданным правилам: 

1.ранжирование строк отдельно в рамках каждого пользователя;

2.ранжирование происходит на основании номера задачи и даты-времени ее решения (если пользователь и задача совпадают, принимаем решение на основании даты-времени, все сортировки по возрастанию).

Создание нового датафрейма df по заданным правилам:

1.остаются только ранги >3;

2.по всем рангам рассчитывается количество уникальных пользователей, у которых встречался данный ранг (столбец amount);

3.добавлен столбец diff, который показывает % динамику между текущим amount и предыдущим (пропуски данных заменены на 0, значения округлены до 2 знака после запятой).

_**Пример выполнения:**_

rnk	amount	diff

0	4	660	0.00

1	5	625	-0.05

2	6	595	-0.05

3	7	569	-0.04

4	8	543	-0.05

_**Анализ данных:**_

1. Абсолютные потери пользователей: с 4-й на 5-ю попытку: 660 → 625 пользователей (потеря 35 человек, -5.3%), c 5-й на 6-ю попытку: 625 → 595 пользователей (потеря 30 человек, -4.8%), с 6-й на 7-ю попытку: 595 → 569 пользователей (потеря 26 человек, -4.4%).

2. Наибольшие относительные потери: самое большое падение: с 4-й на 5-ю попытку (-5%, diff = -0.05), также значительные потери: с 5-й на 6-ю (-5%) и с 8-й на 9-ю (-5%).

Ключевой вывод: большинство пользователей "отваливается" на переходе с 4-й на 5-ю попытку. Психологический барьер: После 4-й неудачной попытки многие пользователи теряют мотивацию.

Рекомендации для платформы: на 4-й попытке стоит добавить дополнительные подсказки, мотивирующие сообщения, возможность посмотреть решение или обратиться за помощью.

После 5-й попытки: предложить альтернативные подходы к решению, дать возможность перейти к следующей задаче и вернуться позже, предоставить более детальный разбор ошибок.

Дополнительные наблюдения: после 11-й попытки падение замедляется (-2%) — остаются самые упорные пользователи.

Общий тренд: постепенное уменьшение количества пользователей с каждой попыткой.

Вывод: чтобы удержать больше пользователей, нужно особенно внимательно работать с теми, кто сделал 4 неудачные попытки — это переломный момент, после которого большинство сдается.
